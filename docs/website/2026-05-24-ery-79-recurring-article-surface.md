# ERY-79 — Recurring article surface (interim, pre-redesign)

Date: 2026-05-24
Owner: Website Engineer
Issue: ERY-79
Parent: ERY-67
Related: ERY-49 (Astro redesign), ERY-54

## What this is

A dedicated, stable **Articles** surface on the current Starlight docs site so marketing can publish recurring biweekly long-form content now — without waiting on the ERY-49 redesign and without burying articles inside `guides/` or `features/`.

## The surface

- **Route:** `/articles/` (index) and `/articles/<slug>/` per article. Locale fallbacks `/nl/articles/` and `/de/articles/` are generated automatically by Starlight until localized articles exist.
- **Source folder:** `website/src/content/docs/articles/`
- **Index page:** `website/src/content/docs/articles/index.md`
- **Template:** `website/src/content/docs/articles/_template.md` (underscore-prefixed → ignored by Astro, never published)
- **Navigation:** an `Articles` group in `website/src/config/sidebar.json` set to `autogenerate: { directory: "articles" }`. New articles appear in the sidebar automatically.

## Publish path marketing should use

This is the named, stable rule that holds until the redesign lands:

1. Copy `articles/_template.md` to `articles/<slug>.md`. The filename becomes the URL slug.
2. Fill in `title` + `description`, write the body, remove the template comment block.
3. Commit. The article publishes at `/articles/<slug>/` and appears in the **Articles** sidebar group with no config edits.

No per-article route or sidebar wiring is needed — that is the point of the autogenerated group.

## Why this does not block the immediate docs path

This is additive. The brief's interim motion (publishing through `guides/` / `features/` and `managed-rollout.mdx`) keeps working unchanged. Marketing can adopt `/articles/` immediately or keep using guides — both build and ship today. No existing route, schema, or navigation entry changed except the additive `Articles` sidebar group.

## Explicit dependency on ERY-49

This surface is intentionally the **lightest viable** option and inherits docs chrome (sidebar, table of contents, docs theme). The ERY-49 Astro redesign is expected to replace it with a purpose-built article/blog template that adds:

- dated/author metadata and an ordered, date-sorted index (interim ordering relies on Git history)
- localized `nl`/`de` articles (interim articles are English-only with auto-generated locale fallbacks)
- marketing-grade layout distinct from docs chrome and richer social-card composition (UXDesigner)

None of the above blocks biweekly publishing now. When ERY-49 ships, migrate the `articles/` markdown into the new template and retire the autogenerated docs group.
